# OpenCV SearchMax 유사 도구 설계

## 1. 목적

Windows 11에서 실행되는 Python/PySide6 데스크톱 앱을 만든다. 사용자는 UI 화면 이미지를 불러와 관심 영역(ROI)을 Train 템플릿으로 지정하거나, 미리 잘라 둔 템플릿 파일을 직접 불러올 수 있다. 앱은 OpenCV의 정규화 상관계수 기반 다중 스케일 템플릿 매칭으로 테스트 이미지에서 템플릿을 찾고, 모든 최종 검출 결과와 점수를 표시한다.

이 도구는 Cognex VisionPro SearchMax 8.x의 사용자 흐름과 핵심 검색 개념을 참고하지만 Cognex 내부 알고리즘의 완전한 복제품은 아니다. 이번 범위는 회전과 원근 변화가 없는 프로그램 UI 이미지 검색에 맞춘다.

## 2. 범위

### 포함

- Windows 11용 Python/PySide6 데스크톱 UI
- 화면 이미지에서 마우스 드래그로 Train ROI 선택
- 잘라 둔 템플릿 이미지 직접 불러오기
- 컬러 또는 그레이스케일 정규화 상관 매칭
- 80~150% 범위의 균일 스케일 검색
- 검색 임계값과 스케일 간격 설정
- 최대 검출 개수 1~100 설정
- 중복 후보 제거와 최종 결과 점수 표시
- 테스트 이미지 불러오기 및 합성 테스트 이미지 생성
- 정답 데이터 기반 자동 평가와 CSV 내보내기
- 프로젝트 설정 저장 및 다시 불러오기

### 제외

- 회전, 비균일 스케일, skew, 원근 변환 검색
- Cognex VPP 파일 호환
- Cognex 점수와 수치적으로 동일한 결과 보장
- 카메라 실시간 취득 및 PLC 연동
- 첫 버전의 PyInstaller 실행 파일 배포

## 3. 기술 선택

- Python 3.11 이상
- PySide6: 데스크톱 UI
- OpenCV: 이미지 입출력, 크기 변경, 템플릿 매칭, 합성
- NumPy: 점수 맵, 노이즈, 좌표 계산
- Pytest: 엔진과 생성기 단위 테스트

기본 매칭 함수는 `cv2.matchTemplate(..., cv2.TM_CCOEFF_NORMED)`이다. 이 방식은 학습 패턴과 실행 이미지의 정규화 상관계수를 사용하는 SearchMax의 핵심 개념과 가깝고, 회전 없는 UI 요소에 적합하다.

## 4. 아키텍처

### UI 계층

이미지 표시, ROI 선택, 설정 편집, 작업 실행, 결과 표와 오버레이를 담당한다. 매칭 알고리즘이나 이미지 합성 규칙을 직접 구현하지 않고 서비스 계층을 호출한다.

### Training 서비스

원본 이미지와 ROI 또는 별도 템플릿 파일을 받아 유효성을 검사하고 Train 모델을 만든다. 원본 컬러 템플릿과 그레이스케일 표현, 크기, 소스 정보를 보관한다.

### Matcher 서비스

Train 모델과 검색 설정을 받아 다중 스케일 NCC 검색을 수행하고 점수순 `MatchResult` 목록을 반환한다. UI와 독립적으로 테스트하고 재사용할 수 있어야 한다.

### Test Generator 서비스

Train 템플릿을 배경 위에 합성하고 정답 박스와 적용 변형을 기록한다. 같은 랜덤 시드와 설정은 같은 테스트 세트를 생성해야 한다.

### Evaluator 서비스

정답 박스와 검출 결과를 비교해 IoU, 위치 오차, 스케일 오차, 성공 여부와 집계 통계를 계산한다.

### Persistence 서비스

프로젝트 JSON, 테스트 메타데이터 JSON, CSV 결과를 읽고 쓴다. 프로젝트 파일에는 이미지 자체 대신 이식 가능한 상대 경로를 우선 기록한다.

## 5. UI 설계

메인 창은 세 개의 설정 영역, 중앙 이미지 뷰어, 하단 결과 영역으로 구성한다.

### Train 영역

- Train 원본 이미지 불러오기
- 템플릿 파일 직접 불러오기
- 이미지 위 ROI 선택 및 Train 실행
- 현재 템플릿 미리보기와 크기 표시
- Train 상태 및 유효성 경고 표시

### Test / Generator 영역

- 검색할 테스트 이미지 한 장 또는 여러 장 불러오기
- 배경 이미지 한 장 또는 여러 장 불러오기
- 테스트 이미지 생성 개수와 랜덤 시드 설정
- 밝기, 대비, 블러, 노이즈 변형 활성화 및 강도 설정
- 생성 이미지 목록과 일괄 실행

### Search Settings 영역

- 매칭 모드: 컬러(기본) 또는 그레이스케일
- 최소/최대 스케일: 기본 80/150%
- 스케일 간격: 기본 2%
- 합격 임계값: 기본 0.80
- 최대 검출 개수: 기본 1, 범위 1~100
- NMS IoU 임계값: 기본 0.30

### 이미지 뷰어

- 확대, 축소, 화면 맞춤, 패닝
- Train 원본에서 마우스 드래그 ROI 선택
- 검색 결과 박스, 중심점, 순위, 점수 표시
- 정답 박스는 초록색, 검출 박스는 빨간색으로 구분
- 결과 표에서 행을 선택하면 해당 박스를 강조

### 결과 영역

최종 검출 결과를 점수 내림차순으로 표시한다. 열은 순위, 점수, X, Y, 너비, 높이, 스케일, 처리 시간을 포함한다. 원시 점수 맵의 모든 픽셀을 결과로 노출하지 않는다. 진단 옵션을 활성화하면 NMS 적용 전 상위 후보를 별도 표에서 확인할 수 있다.

일괄 테스트 요약에는 성공 수/전체 수, 성공률, 평균 IoU, 평균 위치 오차, 평균 스케일 오차, 평균 처리 시간을 표시한다.

## 6. 매칭 알고리즘

1. 설정된 최소 스케일부터 최대 스케일까지 지정 간격으로 스케일 목록을 만든다.
2. 각 스케일로 Train 템플릿 크기를 변경한다. 확대에는 선형 보간, 축소에는 영역 보간을 사용한다.
3. 스케일된 템플릿이 검색 이미지보다 크면 해당 스케일을 건너뛴다.
4. 선택된 컬러 또는 그레이스케일 모드로 `TM_CCOEFF_NORMED` 점수 맵을 계산한다.
5. 점수 맵에서 합격 임계값 이상인 지역 최댓값을 후보로 수집한다. 한 스케일에서 지나치게 많은 후보가 생기지 않도록 진단용 내부 상한을 둔다.
6. 모든 스케일의 후보를 합치고 점수 내림차순으로 정렬한다.
7. IoU 기반 NMS로 같은 인스턴스를 나타내는 중복 후보를 제거한다.
8. 남은 결과 중 `최대 검출 개수`만 반환한다.

각 결과는 점수, 좌상단 좌표, 너비, 높이, 중심 좌표, 스케일을 가진다. 점수 범위는 OpenCV 계산 결과를 0~1 표시 범위로 제한한다. 처리 시간은 한 이미지의 전처리부터 최종 결과 생성까지 측정하며 결과별 공통 실행 시간과 전체 일괄 시간을 구분한다.

## 7. 테스트 이미지 생성

### 입력

- Train 템플릿
- 선택적 배경 이미지 목록
- 출력 크기
- 생성 개수
- 랜덤 시드
- 변형 범위

배경을 제공하지 않으면 단색 또는 단순한 UI 형태가 있는 합성 배경을 만든다. 템플릿은 결과 이미지 경계 안에 완전히 들어오도록 배치한다.

### 변형

- 균일 스케일: 80~150%
- 위치: 유효 영역 내 무작위 좌표
- 밝기와 대비: 기본적으로 원본 식별성을 해치지 않는 약한 범위
- Gaussian blur: 없음 또는 약한 커널
- Gaussian noise: 없음 또는 약한 표준편차

밝기, 대비, 블러, 노이즈는 합성된 전체 이미지에 적용해 템플릿 경계가 부자연스럽게 분리되지 않게 한다. 회전과 원근 변환은 적용하지 않는다.

각 생성 이미지에는 파일명, 정답 박스, 실제 스케일, 변형 파라미터, 랜덤 시드를 담은 JSON 메타데이터를 저장한다.

## 8. 자동 평가

각 정답 인스턴스와 검출 결과는 IoU가 가장 높은 일대일 조합으로 대응한다. 기본 성공 기준은 다음 두 조건을 모두 만족하는 것이다.

- 검출 점수가 사용자가 정한 합격 임계값 이상
- 정답 박스와 검출 박스의 IoU가 0.50 이상

평가 결과에는 성공 여부, 최고 대응 IoU, 중심 위치 오차(픽셀), 스케일 오차(%), 검출 점수, 처리 시간을 포함한다. 사용자가 불러온 외부 테스트 이미지는 정답 정보가 없으므로 점수와 좌표만 표시하고 자동 성공률에는 포함하지 않는다.

## 9. 저장 형식

프로젝트 JSON은 다음을 포함한다.

- 스키마 버전
- Train 이미지 또는 템플릿 경로와 ROI
- 검색 설정
- 테스트 생성 설정
- 최근 테스트 이미지 및 배경 경로

생성 테스트 이미지는 선택한 출력 폴더에 PNG로 저장하고 정답은 JSON으로 저장한다. 검색 및 평가 결과는 화면에서 확인하고 CSV로 내보낼 수 있다.

## 10. 오류 처리

- 지원하지 않는 이미지 형식 또는 디코딩 실패: 파일명과 원인을 표시한다.
- ROI가 이미지 밖이거나 너무 작음: Train을 거부하고 최소 크기를 안내한다.
- 템플릿의 표준편차가 거의 0임: 안정적인 정규화 상관 점수를 만들 수 없음을 경고한다.
- 유효 스케일이 없음: 템플릿과 검색 이미지 크기 및 설정 범위를 안내한다.
- 후보가 없음: 정상적인 `0 results` 상태로 처리한다.
- 설정값 범위 오류: 해당 입력 필드에서 즉시 검증한다.
- 파일 저장 실패: 대상 경로와 운영체제 오류를 표시하고 앱 상태를 유지한다.
- 일괄 작업 중 개별 파일 실패: 나머지 파일은 계속 처리하고 실패 목록을 요약한다.

## 11. 검증 전략

### 단위 테스트

- 정확한 크기와 위치에서 단일 템플릿 검출
- 80%, 100%, 150% 스케일 검출
- 임계값 경계 동작
- 최대 검출 개수 제한
- 스케일 간 중복 후보에 대한 NMS
- IoU와 평가 매칭 계산
- 랜덤 시드에 따른 생성 재현성
- 너무 큰 템플릿과 저분산 템플릿 오류

### 통합 테스트

- ROI Train부터 검색 결과 표시까지의 흐름
- 직접 템플릿 불러오기부터 일괄 검색까지의 흐름
- 테스트 세트 생성, 저장, 다시 불러오기, 평가
- 프로젝트 저장 및 복원
- CSV 결과 내보내기

### Windows 수동 확인

- Windows 11, 100/125/150% 디스플레이 배율에서 UI 레이아웃
- 한글 경로 및 한글 파일명 이미지 입출력
- 큰 스크린샷에서 UI 응답성과 진행 상태 표시
- 취소 가능한 일괄 검색과 오류 메시지

## 12. 완료 기준

- 두 가지 Train 방식이 모두 동작한다.
- 80~150% 범위에서 생성한 템플릿을 검출하고 좌표와 점수를 반환한다.
- 최대 검출 개수 설정이 1~100 범위에서 정확히 적용된다.
- 모든 최종 검출 결과가 점수순 표와 오버레이에 나타난다.
- 같은 시드로 같은 테스트 이미지와 메타데이터가 생성된다.
- 생성 테스트 세트의 결과를 자동 평가하고 CSV로 내보낼 수 있다.
- 핵심 단위 및 통합 테스트가 통과한다.
- Windows 11 실행 절차가 문서화되어 있다.
